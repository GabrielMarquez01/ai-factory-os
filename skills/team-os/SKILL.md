# 👥 Team OS

> **Analogía:** Un solopreneur es como un restaurante de un solo cocinero — rápido, con personalidad, pero con límite de mesas. Contratar sin sistema es como agregar cocineros sin recetas escritas — el caos aumenta antes que la capacidad.
>
> El Team OS son las recetas escritas que permiten escalar sin perder calidad.

## La Señal de Que Es Hora de Contratar

No contratas porque quieres crecer. Contratas cuando el crecimiento está limitado por tu tiempo.

**Las 4 señales claras:**

```
1. CUELLO DE BOTELLA IDENTIFICADO
   Hay una tarea específica que solo tú puedes hacer y que limita el negocio
   Si esa tarea se delegara, el negocio crecería
   
2. RETORNO POSITIVO CLARO
   Lo que ganas por tener esa tarea delegada > lo que pagas por delegarla
   Ejemplo: si delegar soporte te libera 20h/semana y puedes usar esas 20h
   para generar $X más → el contrato se paga solo
   
3. PROCESO DOCUMENTADO
   Puedes escribir exactamente cómo hacer esa tarea en 30 minutos
   Si no puedes documentarlo → aún no estás listo para delegar
   
4. FLUJO DE CAJA POSITIVO
   Tienes al menos 3 meses de sueldo en reserva antes de contratar
   Contratar en modo "necesito ayuda urgente" es la peor posición de negociación
```

## El Framework: Agente IA vs Asistente Humano vs Empleado

Antes de contratar una persona, evalúa si un agente puede hacer el trabajo:

```
┌─────────────────────────────────────────────────────────┐
│                  ÁRBOL DE DECISIÓN                      │
├─────────────────────────────────────────────────────────┤
│ ¿La tarea es repetible y basada en reglas?              │
│   → Sí: ¿Ya existe un agente/skill para esto?          │
│           → Sí: usar el agente                         │
│           → No: construir el agente (ver agent-architect)│
│   → No: requiere juicio humano                         │
│           ↓                                            │
│ ¿Se necesita creatividad o relación con personas?       │
│   → No: asistente virtual (freelance remoto)           │
│   → Sí: colaborador de confianza (empleado o socio)    │
└─────────────────────────────────────────────────────────┘
```

## Los 3 Tipos de Colaboradores (en Orden de Compromiso)

### Tipo 1: Freelancer por Proyecto

**Cuándo:** necesitas una habilidad específica una sola vez o esporádicamente.

```
Ejemplos: diseñador para el logo, desarrollador para una feature específica,
          fotógrafo para el lanzamiento de producto

Plataformas: Workana (LatAm), Upwork, 99designs, LinkedIn

Contrato mínimo:
- Descripción exacta del entregable (no "diseña algo bonito")
- Fecha de entrega específica
- Revisiones incluidas (máximo 2-3)
- Precio fijo, no por hora (para proyectos definidos)
```

### Tipo 2: Asistente Virtual Recurrente

**Cuándo:** hay tareas repetibles que consumen tiempo pero no requieren tu expertise único.

```
Ejemplos: administración de email, publicación de contenido,
          respuesta a comentarios, investigación de mercado,
          data entry, coordinación de proveedores

Cómo encontrar: Remoto en LatAm (México, Colombia, Argentina) → mejor relación
                calidad/costo para negocios en español

Lo más importante al contratar un AV:
- 1 semana de prueba pagada con tareas reales (no "prueba gratis")
- Manuales escritos de cada tarea (el manual es más importante que el AV)
- Métricas claras de éxito (¿cómo sé si está haciendo bien el trabajo?)
```

### Tipo 3: Socio o Colaborador de Confianza

**Cuándo:** necesitas a alguien que piense contigo, no solo ejecute.

```
La diferencia con el AV: el socio propone, mejora, toma iniciativa
No hace "lo que le dices" — hace lo que "debe hacerse"

Cómo encontrar: comunidad, network, no en plataformas de freelancers

Antes de hacer socio a alguien:
- Trabaja con ellos en un proyecto pequeño primero (nunca commitment grande sin prueba)
- Define expectativas de roles y compensación por escrito desde el inicio
- Alinea valores y forma de trabajar (las diferencias aquí destruyen sociedades)
```

## El Manual de Operaciones: La Clave de la Delegación

Un manual de operaciones es la documentación de cómo funciona tu negocio.

Sin manual: cuando el AV se va, el conocimiento se va con él.
Con manual: cualquier persona competente puede hacer el trabajo con el manual.

**Estructura de un Manual de Tarea:**

```markdown
# Manual: [Nombre de la Tarea]

## Objetivo
[Qué resultado produce esta tarea y por qué importa]

## Frecuencia
[Diario / Semanal / Cuando ocurre X evento]

## Herramientas necesarias
- [Herramienta 1] — acceso en [dónde]
- [Herramienta 2] — credenciales en [gestor de contraseñas]

## Proceso paso a paso
1. [Paso 1 con captura de pantalla si es necesario]
2. [Paso 2]
3. ...

## Criterios de calidad (¿cómo sé que está bien hecho?)
- [Criterio 1]
- [Criterio 2]

## Qué hacer si algo sale mal
- [Situación frecuente 1] → [cómo resolverla]
- [Situación frecuente 2] → [cómo resolverla]
- Si no sabes qué hacer: [a quién escalar y cómo]

## Tiempo estimado
[X minutos/horas por [frecuencia]]
```

## Comunicación Asíncrona: La Base del Equipo Remoto

Un equipo distribuido necesita comunicación que no dependa de estar en línea al mismo tiempo:

```
REGLAS DE COMUNICACIÓN ASÍNCRONA:

Loom (loom.com) para explicaciones complejas → video < 5 minutos
Notion para documentación permanente → todo lo que debe sobrevivir al tiempo
Linear para tareas y proyectos → con fecha, dueño y criterio de éxito
Slack/WhatsApp solo para urgente → urgente = afecta a un cliente ahora mismo

La regla de oro: si puedes esperarlo hasta mañana → no es urgente
```

## Gates de Comunicación con el Equipo

El mismo sistema de gates que usas con agentes aplica con personas:

```
Cada colaborador tiene un "nivel de autonomía":
L0: te consulta antes de ejecutar cualquier cosa
L1: ejecuta y reporta al final del día
L2: ejecuta y reporta si algo sale diferente a lo esperado

Meta: todos en L1 o L2 en sus áreas de expertise
Nunca: L2 para decisiones que afectan dinero, clientes, o estrategia
```

## La Regla del Team OS

> Contrata lento. Documenta primero. Delega gradualmente.
>
> El costo de una mala contratación no es solo el sueldo —
> es el tiempo perdido en re-hacer el trabajo y el daño a la moral.
>
> El mejor momento para escribir el manual es ANTES de contratar,
> no después.
