# 🎯 Funnel de Infoproducto

> **Analogía:** Un carpintero que construye una silla de lujo y luego la saca a vender no sabe si alguien la quiere al precio que pide. Un carpintero que toma pedidos primero y cobra adelantado, luego construye — ese nunca tiene inventario sin vender.
>
> La preventa no es un truco de marketing. Es el test de realidad más honesto que tienes.

## El Principio Más Importante

**No construyas lo que no has vendido.**

El error más común del fundador independiente:
1. Tiene una idea ✓
2. Pasa semanas construyendo el producto ✓ ← Error aquí
3. Lo lanza ✓
4. Nadie compra ✓
5. Se pregunta qué salió mal ✓

El sistema correcto:
1. Tiene una idea
2. Valida en 72 horas si hay demanda real
3. Pre-vende (cobra antes de construir)
4. Construye lo que ya pagaron
5. Entrega con fecha comprometida
6. Itera con feedback real

## Los 4 Componentes del Funnel

### Componente 1: La Landing Page de Preventa

Una sola página. Un solo objetivo: conseguir que paguen.

**Estructura AIDA (Atención, Interés, Deseo, Acción):**

```
[HEADLINE] — Qué problema resuelves en 6-8 palabras
           Ejemplo: "Aprende [resultado deseado] sin [obstáculo principal]"

[PROBLEMA] — El dolor que siente tu avatar en sus propias palabras
             Muestra que lo entiendes antes de ofrecer la solución

[SOLUCIÓN] — Qué es tu producto y qué hace exactamente
             Evita el lenguaje genérico ("te transformará la vida")

[PRUEBA]   — Por qué confiar en ti
             Tu historia, tu experiencia, tus resultados propios

[OFERTA]   — El precio, qué incluye, por qué ahora
             Específico: "1 ebook de 47 páginas + plantilla descargable"

[CTA]      — Botón de acción con verbo específico
             "Quiero mi guía" > "Comprar" > "Enviar"

[OBJECIONES] — Las 3 dudas que tiene antes de pagar
              Respóndelas antes de que las pregunte

[CTA FINAL] — Segundo botón al pie de la página
```

**Lo que NO incluye una landing de preventa:**
- Menú de navegación (el usuario no debe poder "escapar")
- Múltiples ofertas o precios que causen parálisis
- Videos de 10+ minutos antes de la oferta
- Información que no ayude a decidir la compra

### Componente 2: El Mecanismo de Cobro

Para una preventa, el cobro debe ser inmediato, seguro, y simple.

| Plataforma | Dificultad de setup | Fee por venta | Mejor para |
|-----------|--------------------|--------------| ----------|
| **Gumroad** | Muy fácil (5 min) | 10% | Validar rápido, primeras ventas |
| **Polar.sh** | Fácil (30 min) | 4% | Infoproductos, SaaS |
| **Hotmart** | Medio (1-2h) | Variable | LatAm, cursos con afiliados |
| **Stripe** | Moderado (KYC) | 2.9% + $0.30 | Volumen alto, mayor control |

> **Para una primera preventa de validación:** usa la plataforma de menor fricción aunque el fee sea más alto. Ganar tiempo > ahorrar 6% en 5 ventas.

### Componente 3: Emails Post-Compra

Una vez que pagan, comienza la relación. Ver skill `email-nurturing-postcompra` para el sistema completo.

**El email más importante: confirmación inmediata (< 5 minutos después del pago)**

```
ASUNTO: Recibí tu pedido ✓ — [Nombre del producto]

CUERPO:
- Confirmar que el pago fue exitoso
- Expectativa clara: cuándo recibirán el producto
- Qué pueden hacer mientras tanto (acceso anticipado, comunidad)
- Tu contacto directo para cualquier duda
- Agradecimiento genuino (eres un negocio de una persona — hazlo personal)
```

### Componente 4: La Entrega

**Para ebooks / guías / plantillas (digitales):**
- PDF entregado por email o link de descarga
- Página de "Gracias" con acceso inmediato
- Si está en construcción: dar fecha específica y cumplirla

**Para productos en construcción durante la preventa:**
- Comunicación semanal de avance (muestra que estás construyendo)
- Acceso por módulos según se vayan terminando
- Fecha de entrega comprometida públicamente = presión positiva para cumplir

## El Gate de Validación

Antes de invertir tiempo en construir, define tu meta:

```
GATE DE VALIDACIÓN:

Meta:    [N] personas pagan en [X días]
Acción si SÍ: construir el producto completo
Acción si NO: devolver el dinero + entrevistar a quienes estuvieron cerca de comprar

Ejemplo:
"Si 5 personas pagan $299 MXN en 72 horas → construyo el ebook"
"Si no llego a 5 → reembolso total + encuesta de por qué no compraron"
```

**¿Por qué 5 y no 50?** Para un solopreneur con audiencia pequeña, 5 preventas en 72h con comunicación personal es señal de demanda real. Ajusta el número a tu tamaño de audiencia actual.

## La Campaña de 72 Horas

El patrón para ejecutar una preventa rápida:

```
HORA 0  → Publicar landing page
          Anunciar en TODOS tus canales activos
          Mensaje: "Estoy creando [producto]. Si compras hoy, precio de fundador."
          Precio de fundador: 30-50% menos que el precio final

HORA 24 → Recap del avance + recordatorio a quienes abrieron el email
          Actualización honesta: "Ya van [N] personas, faltan [X] para el gate"

HORA 48 → "Quedan 24 horas para el precio de fundador"
          Comparte una muestra del contenido (da confianza)

HORA 72 → "Cierra hoy" + mensaje directo a los más interesados
          DM / mensaje personal a quienes preguntaron pero no compraron

HORA 73+ → Evaluar gate:
           ✅ Llegaste: comunicar a compradores que se construye
           ❌ No llegaste: reembolsar + preguntar qué frenó
```

## Anatomía de una Oferta Que Convierte

```
[NOMBRE DESCRIPTIVO]
No: "Guía Maestra" (genérico)
Sí: "Guía de [resultado específico] para [avatar específico] en [tiempo realista]"

+ [PRECIO ANCLA si tienes uno] ~~$XXX~~ (tachado, con razón real)
+ [PRECIO ACTUAL] $XX — Precio de fundador hasta [fecha]
+ [LO QUE INCLUYE] (bullets cortos, beneficios > características)
+ [GARANTÍA si la tienes] — reduce el riesgo percibido del comprador
+ [URGENCIA real] — si dices "solo hoy", que sea verdad
+ [BOTÓN] con verbo de acción: "Quiero mi guía" > "Comprar ahora" > "Agregar al carrito"
```

## Qué Aprender Cuando No Compran

Si la preventa no llega al gate, no es fracaso — es información valiosa.

**Diagnóstico rápido:**

```
¿La gente llegó a la landing?
→ Sí pero no compraron: problema en la oferta, precio, o confianza
→ No: problema de distribución (no los alcanzaste o no les interesó el tema)

¿Abrieron el email / vieron el post?
→ Sí pero no hicieron click: el headline no enganchó
→ No: problema de alcance o momento inadecuado

¿Llegaron al botón de pago y se fueron?
→ Problema de precio, fricción técnica, o falta de garantía
```

**La pregunta más valiosa para hacer a quienes casi compraron:**
"¿Qué te frenó?" — una sola pregunta abierta, sin opciones.

> Vende primero. Construye después.
>
> Si nadie compra cuando el producto no existe,
> nadie comprará cuando sí exista — pero habrás invertido semanas.
>
> La preventa no es un truco: es el método.
